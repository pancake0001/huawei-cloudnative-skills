# Workflow

## Kubernetes Stdout Logs

1. Identify `region`, `cluster_id`, `namespace`, `pod_name`, and optional `container`.
2. Use `huawei_get_pod_logs` for direct Kubernetes stdout/stderr retrieval.
3. Use `tail_lines` for focused recent output; use `previous=true` for a previous terminated container.
4. Summarize errors, warnings, stack traces, restarts, or repeated messages.

## CCE Application Logs Through LTS

1. Use `huawei_get_cce_logconfigs` to inspect LogConfig collection rules.
2. Use `huawei_get_application_logconfigs` to map an app/workload to matched LogConfig policies and their LTS `log_group_id` / `log_stream_id`.
3. Pick the desired policy from `matched_streams`, then pass `logconfig_name` or `policy_name` to `huawei_query_application_recent_logs` for recent logs, or `huawei_query_application_logs` for explicit time windows.
4. Prefer the labels generated by the tool. It adds `clusterId`, `appName`, `nameSpace`, and when a policy is specified, `logconfig`.
5. Use stdout policies for standard output logs and `container_file` policies for file logs collected from configured paths.
6. Use `huawei_create_cce_logconfig` when a required stdout or container file LogConfig is missing. Call it without `confirm=true` first, inspect the generated `request_body`, then call with `confirm=true` only after confirmation.
7. Use `huawei_delete_cce_logconfig` only when the user explicitly asks to remove a LogConfig. Call it without `confirm=true` first, inspect the returned target summary, then call with `confirm=true` only after confirmation.
8. Use `huawei_query_cce_audit_logs` for Kubernetes audit questions. It is pure keyword search over audit log content; all convenience parameters are converted into keywords and no parsed-field filtering is applied.
9. Prefer explicit audit `log_group_id` and `log_stream_id` when known. If omitted, the audit tool attempts to discover LTS streams by names containing audit/apiserver terms.
10. Use `auto_paginate=true` when more than one LTS page is needed. `limit` controls page size and `max_pages` caps the number of pages fetched.
11. Use `huawei_analyze_application_logs` for time-window abnormality analysis. It queries the matched application logs, detects exception/error/fatal/timeout/OOM patterns and HTTP 5xx status codes, then returns abnormal ratio, log rates, first/last abnormal time, observed recovery time, incident windows, top patterns, status-code distribution, and samples.
12. If a user asks for an abnormality ratio, do not set `keywords` unless they explicitly want a keyword-scoped ratio. Keyword filtering changes the denominator to only matched logs.
